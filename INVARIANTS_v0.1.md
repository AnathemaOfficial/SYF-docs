# Invariants

**Canonical — v0.1 (SEALED)**

---

## Purpose

Formally list the **fundamental invariants** of the SYF / Anathema system.

These invariants define **what is impossible**, not what is desirable.

## Canonical Invariants

### I-1 — Separation of Intention and Action

No intention can produce an action without interposition.

- Intent and capacity are structurally separated
- The interposition layer cannot be bypassed
- Direct action is architecturally impossible

### I-2 — Fail-Closed by Construction

Any ambiguity results in denial.

- Unknown states → denial
- Uncertain conditions → denial
- Missing validation → denial
- Default state is always: **blocked**

### I-3 — No Externalities

No external dependency (oracle, human, market).

- No external truth sources
- No time-dependent validation
- No price feeds
- No consensus mechanisms
- No network dependencies for core function

### I-4 — No Governance

No human or collective decision.

- No voting mechanisms
- No committee overrides
- No democratic amendments
- No human-in-the-loop for core decisions

### I-5 — Determinism

Same input → same result, always.

- Reproducible behavior
- Auditable execution
- No randomness in core logic
- No state-dependent variation

### I-6 — Present-Only Enforcement

No projection, no anticipation.

- No predictive enforcement
- No future state consideration
- No historical pattern matching
- Validation happens **now** or not at all

### I-7 — Law Supremacy

The law prevails over any implementation.

- Implementation cannot override law
- Product cannot modify invariants
- Code does not define truth
- Canon is the sole reference

### I-8 — Automatic Revocation on Violation

Any violation invalidates usage immediately.

- No grace period
- No warning required
- No arbitration
- Structural, not contractual

### I-9 — Read-Only Canon

Canons are immutable.

- No amendments
- No versioning of core law
- No interpretation drift
- What is sealed remains sealed

### I-10 — Non-Negotiability

No invariant is amendable.

- No exceptions
- No temporary suspensions
- No contractual overrides
- No jurisdictional variations

## Global Effect

These invariants collectively define:

- The **boundary of possibility**
- The **structure of safety**
- The **nature of compliance**

A system is SYF-compliant if and only if all ten invariants are satisfied simultaneously and continuously.
